unidade FR

digamos q tu coloque 3 colunas com 1 FR cada, isso vai dividr a tela em 3 olunas,
mas isso quer dizer que 1 Fr tem aquele tam real ? NAO ! isso vai sempre depender do tamanho do grid.
por exemplo se tu colocar um tam fixo pro grid de 400px, essas 3 colunas de 1FR vai se dividir dentro desse espaco

ele se divide proporcionalmente dentro do espaco

            grid-template-columns: 1fr 1fr 1fr;

agr digamos que colocamos 2fr, esse valor de 2fr sera o dobor do valor do 1fr q tiver com ele, independente da posicao, eles vao ser recalculados para se ajustarem

assim se vc por 3fr sera a mesma logica.

            grid-template-columns: 1fr 2fr 1fr;

lembrando q coloquei um valor fixo, mas que n use um valor fixo os fr vap se adaptar no tam de tela q vc tiver.

grid-auto-rows

pode ser usado de muitas formas, mas vms usar de uma forma basica para comecar, digamos que quero q todas tenham 200px

    
        grid-auto-rows:200px;

porem nos colocamos um texto mt grande, e ele vai acabar saindo dessa linha auto que criamos

para solocionar isso, usamos o nosso minmax

minmax

    
        Entao dessa forma, eu limito o tam das linhas de acordo com o conteudo delas.
        entao se o nosso elemento 5, tiver um texto q cubra 200px da row,ele automaticamente, vai modificar todas as outras desse modo na mesma.
        grid-auto-rows: minmax(50px, 200px)

        Porem, eu nao quero que todas fiquem alteradas.Entao dessa forma eu limito o tam min de 500px
        porem, o tam max ficara automatico, ou seja sera de acordo com o tam que eu tiver ocupado dentro do elemento, e obviamente toda linha acompanhara.
        grid-auto-rows: minmax(50px, auto);

Mas caso prefira pode usar o minmax dentro do proprio grid-template-columns 

    
        grid-template-columns: repeat (3, minmax(100px, 300px));

Um ponto importante e, se eu defini aqueles valores e, coicide de eu ter muitos elementos, ele seguira
a regra na mesma, mesmo que tenha que criar uma barra de rolagem.
